# 文章创作助手

将任意想法、链接或素材转化为符合目标平台风格的高质量文章。

## 使用方式

```
写文章 <内容来源> [平台] [类型] [风格] [长度]
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `内容来源` | 想法描述、文章链接、文档路径 | "AI工具对设计师的影响"、"https://xxx.com/article"、"/path/to/doc.md" |
| `平台` | 发布渠道 | 小红书、公众号、知乎、X/Twitter、掘金、CSDN、B站专栏、头条 |
| `类型` | 文章体裁 | 技术教程、经验分享、时事评论、产品评测、故事/小说、干货清单、热点解读 |
| `风格` | 语气风格 | 轻松幽默、专业严谨、亲切随和、犀利吐槽、温暖治愈、硬核技术 |
| `长度` | 篇幅控制 | 短篇(300-500字)、中篇(800-1500字)、长文(2000字+) |

### 示例指令

```
写文章 "AI工具对设计师的影响" 小红书 经验分享 轻松幽默 中篇
写文章 https://example.com/tech-news 公众号 热点解读 犀利吐槽 长文
写文章 "我学编程的三年踩坑史" 掘金 技术教程 亲切随和 长文
写文章 "/root/docs/idea.md" 知乎 干货清单 专业严谨 中篇
```

## 核心能力

### 1. 内容来源处理

**想法/主题 → 文章**
- 根据主题自动扩展大纲
- 补充背景知识和案例
- 构建有逻辑的文章结构

**链接 → 文章**
- 使用 web_fetch 获取链接内容
- 提取核心观点并重构
- 用自己的语言重新组织（避免复制）

**文档 → 文章**
- 读取本地文档内容
- 按需求重组和扩写
- 转换为目标平台格式

### 2. 反AI检测机制（重要）

写作时必须执行以下策略，使内容具有人文温度：

**语言层面**
- ❌ 避免："首先...其次...最后..."、"综上所述"、"值得注意的是"
- ❌ 避免：过度对称的排比句
- ❌ 避免：每段都整齐划一的长度
- ✅ 使用：口语化连接词（"说实话"、"其实"、"你懂的"）
- ✅ 使用：不完美的句式，偶尔来点碎碎念
- ✅ 使用：个人经历和具体细节

**结构层面**
- ❌ 避免：过于工整的123分点
- ❌ 避免：每个段落都遵循"观点-论据-总结"公式
- ✅ 允许：某些段落只说一半，留有余味
- ✅ 允许：偶尔跑题再拉回来
- ✅ 允许：长短句交错，甚至来个单字段

**情感层面**
- 加入真实的情绪波动（困惑、惊喜、无奈）
- 使用第一人称"我"的视角叙述
- 加入具体的场景描写（时间、地点、感官细节）

### 3. 平台风格适配

根据目标平台自动调整：

| 平台 | 核心特征 | 特殊要求 |
|------|----------|----------|
| 小红书 | 种草感、emoji、分段短 | 标题党、多用「」、结尾call to action |
| 公众号 | 深度、故事性、排版友好 | 适合长文、金句突出、引导转发 |
| 知乎 | 专业、逻辑、信息密度 | 引用数据、多角度分析、避免软广感 |
| X/Twitter | 简短、观点鲜明、可传播 | 280字限制、多用@ #、引发讨论 |
| 掘金 | 技术、实操、代码友好 | 代码块、命令行、步骤清晰 |
| B站专栏 | 年轻、玩梗、二次元元素 | 适合追热点、互动性强 |
| 头条 | 标题吸睛、信息量大 | 悬念式标题、适合中老年读者 |

详细风格指南见 `references/platform-styles.md`

### 4. 自动配图建议

在需要插图的位置插入标记：

```markdown
![建议配图：一张展示AI绘画界面的截图，画面中有设计师正在使用Midjourney，色调偏暖](配图占位)

文章内容...

![建议配图：一张对比图，左侧是传统设计流程，右侧是AI辅助流程，用箭头连接](配图占位)
```

配图建议应包含：
- 画面内容描述
- 色调/风格建议
- 构图要点
- 可使用AI绘画工具生成（如需要）

## 输出格式

统一输出 Markdown 格式：

```markdown
# 文章标题

> 副标题/导语（如适用）

![建议配图：xxx](配图占位)

正文内容...

---

**发布建议：**
- 最佳发布时间：xxx
- 话题标签：#xxx #xxx
- 互动引导：xxx

**配图清单：**
1. xxx - 用于段落xxx
2. xxx - 用于段落xxx
```

## 执行流程

1. **解析需求**：确认内容来源、平台、类型、风格、长度
2. **获取素材**：如果是链接/文档，先读取内容
3. **确定角度**：根据平台和类型确定切入角度
4. **构建大纲**：设计文章结构（避免太工整）
5. **撰写正文**：逐段写作，执行反AI检测策略
6. **插入配图建议**：在适当位置标注配图需求
7. **添加发布建议**：根据平台特性给出运营建议
8. **输出成品**：统一Markdown格式

## 注意事项

- 如用户未指定某项参数，询问或根据内容自动判断
- 涉及链接时，先使用 web_fetch 获取内容
- 保持内容的原创性，不要直接复制原文
- 配图建议要具体，方便后续生成或寻找
- 时刻注意规避AI写作的典型痕迹
