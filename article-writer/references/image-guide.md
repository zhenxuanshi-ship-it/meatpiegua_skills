# 配图建议规范

## 配图原则

1. **内容相关**：图片必须服务文章内容
2. **视觉吸引**：首图最重要，决定点击率
3. **平台适配**：不同平台对图片尺寸要求不同
4. **版权安全**：建议用AI生成或明确可商用

## 各平台图片规格

| 平台 | 首图尺寸 | 配图尺寸 | 特殊要求 |
|------|----------|----------|----------|
| 小红书 | 3:4 或 1:1 | 同首图 | 可多张拼图 |
| 公众号 | 900×383 | 宽度900 | 封面图重要 |
| 知乎 | 无强制 | 宽度690 | 可配信息图 |
| X | 1200×675 | 同首图 | 单图或多图 |
| 掘金 | 无强制 | 宽度800 | 技术截图为主 |
| B站专栏 | 960×600 | 同首图 | 二次元风格 |
| 头条 | 无强制 | 宽度≥400 | 新闻配图感 |

## 配图类型

### 1. 封面/首图
- **作用**：吸引点击
- **风格**：根据平台调性
- **元素**：标题文字、视觉主体、品牌色

### 2. 内容插图
- **截图**：软件界面、操作步骤
- **示意图**：流程、对比、架构
- **信息图**：数据可视化
- **氛围图**：情绪渲染

### 3. 表情包/梗图
- 适合：小红书、B站、公众号
- 作用：缓解阅读疲劳，增加互动

## 配图建议格式

在文章中插入配图占位：

```markdown
![建议配图：
- 画面内容：一个人坐在电脑前，屏幕上显示代码，表情困惑
- 色调：暖色调，营造深夜加班氛围
- 风格：扁平插画或真实照片
- 尺寸：900×600
- 生成提示：A programmer sitting at desk at night, confused expression, 
  warm lighting, coding on screen, flat illustration style
](配图占位)
```

## 配图标记示例

### 技术教程类
```markdown
步骤1：安装Node.js

![建议配图：
- 内容：Node.js官网下载页面截图
- 标注：用红框标出LTS版本下载按钮
- 说明：圈出当前最新稳定版](配图占位)
```

### 经验分享类
```markdown
我当时就是在这个界面卡住的...

![建议配图：
- 内容：报错信息截图
- 标注：用箭头指向关键错误代码行
- 说明：真实报错界面，不要美化](配图占位)
```

### 种草/推荐类
```markdown
这就是我要推荐的神器👇

![建议配图：
- 内容：产品实拍或使用场景
- 风格：小红书风，ins风
- 元素：可加入手持、桌面摆放
- 光线：自然光，明亮温暖](配图占位)
```

### 数据对比类
```markdown
来直观感受一下效果：

![建议配图：
- 类型：对比信息图
- 左侧：传统方式，用时10小时，效果一般
- 右侧：新方法，用时2小时，效果显著
- 风格：简洁商务，蓝绿配色](配图占位)
```

### 提示词模板

**扁平插画**
```
Flat illustration of [主题], minimal style, 
pastel colors, clean background, vector art
```

**真实场景**
```
Realistic photo of [场景], natural lighting, 
shallow depth of field, professional photography
```

**信息图表**
```
Infographic showing [数据], modern design, 
clean typography, blue and white color scheme
```

**氛围图**
```
[场景] atmosphere, cinematic lighting, 
moody tones, wide angle, 8k quality
```
